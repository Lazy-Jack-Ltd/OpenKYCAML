# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in the OpenKYCAML schema,
documentation, or tools, please report it responsibly.

**Do not open a public issue for security vulnerabilities.**

Instead, please email the maintainers or use
[GitHub's private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)
feature on this repository.

## What to Include

- Description of the vulnerability
- Steps to reproduce (if applicable)
- Affected files or components
- Potential impact

## Response Timeline

- We will acknowledge receipt within **48 hours**.
- We aim to provide a fix or mitigation within **7 days** for critical issues.
- We will coordinate disclosure with the reporter.

## Scope

This policy covers:

- The JSON Schema definition (`schema/`)
- Validation tools (`tools/`)
- Example files (`examples/`)
- Documentation (`docs/`)
