# Roadmap

This document outlines the public roadmap for the OpenKYCAML Open Schema
Standard. Priorities may shift based on community feedback and regulatory
developments.

## Current (v1.0)

- ✅ Core schema: natural person, legal entity, travel rule, VC wrapper,
  risk assessment
- ✅ IVMS 101 alignment for Travel Rule
- ✅ W3C Verifiable Credential support
- ✅ Python and JavaScript validators
- ✅ Curated examples and public documentation

## Next (v1.1 — planned)

- [ ] Extended address format (structured sub-fields for IVMS 101
  geographicAddress)
- [ ] Tax identification number support (TIN, VAT)
- [ ] Enhanced name structures (IVMS 101 name identifiers)
- [ ] Additional example: multi-party transaction
- [ ] JSON-LD context file for Verifiable Credential issuance

## Future (v2.0 — exploratory)

- [ ] Modular schema architecture (separate sub-schemas for each section)
- [ ] OpenAPI 3.1 integration for API-first usage
- [ ] Selective disclosure profiles for privacy-preserving scenarios
- [ ] Additional regulatory mappings (MiCA, UK FCA, MAS)
- [ ] Multilingual schema labels and descriptions
- [ ] Schema registry and versioned distribution via CDN

## How to Influence the Roadmap

- Open a [GitHub Issue](https://github.com/Lazy-Jack-Ltd/OpenKYCAML/issues)
  with a feature request or use case.
- Reference the regulatory standard or business need driving the request.
- Community demand and regulatory urgency help prioritize items.
